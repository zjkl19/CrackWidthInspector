# 项目技术说明

## 1. 项目目标

本项目用于对混凝土表面裂缝图像进行自动识别、宽度测量和结果导出，目标是形成一个可以直接交付客户使用的 Windows 桌面工具。当前交付形态包括：

- 命令行处理脚本
- `PySide6` 图形界面
- `PyInstaller` 目录版可执行程序

## 2. 技术路线

项目采用“传统图像处理 + 预训练深度边缘先验”的混合方案，不依赖自训练数据集。核心流程如下：

### 2.1 图像预处理

- 彩色图转灰度图
- `CLAHE` 对比度增强
- 中值滤波抑制噪声

### 2.2 裂缝候选提取

- 使用形态学黑帽变换突出暗线状裂缝
- 对黑帽响应进行归一化和高斯平滑
- 使用 `Otsu` 自动阈值获得候选裂缝区域

### 2.3 深度边缘先验约束

- 采用预训练 HED 模型生成边缘响应图
- 选择高响应边缘区域
- 计算候选像素到最近边缘的距离
- 仅保留满足几何一致性的候选裂缝像素

这一步可以明显减少孔洞、污渍、粗糙纹理等背景误检。

### 2.4 骨架提取与宽度估计

- 对二值裂缝掩膜执行骨架化
- 优先使用 `cv2.ximgproc.thinning`
- 如果当前 OpenCV 版本不带 `ximgproc`，回退到 Zhang-Suen 细化算法

宽度估计采用欧氏距离变换：

- 先对裂缝掩膜做 `distanceTransform`
- 骨架点宽度 = 2 × 到边界的距离
- 再乘以 `scale(mm/px)` 换算为毫米

### 2.5 主路径与沿程分析

- 将骨架像素按 8 邻域构造成无向加权图
- 通过两次 Dijkstra 近似提取最长主路径
- 在主路径上进行等距采样
- 导出宽度沿长度方向变化的剖面数据

## 3. 代码结构

### 3.1 核心处理脚本

文件：[crack_width_inspector.py](D:/PythonProjects/CrackWidthInspector/crack_width_inspector.py)

职责：

- HED 模型加载
- 裂缝分割
- 骨架提取
- 宽度计算
- 主路径分析
- 批处理与输出导出
- CLI 入口

新增的工程化能力：

- `process_images(...)` 作为 GUI 可复用入口
- 支持目录批处理时保留输入目录层级，避免同名文件互相覆盖
- 支持打包后从 `PyInstaller` 运行目录中定位模型资源

### 3.2 图形界面

文件：[crack_width_inspector_gui.py](D:/PythonProjects/CrackWidthInspector/crack_width_inspector_gui.py)

职责：

- 选择单图或文件夹输入
- 配置输出目录、比例尺、采样点数
- 后台线程调用核心处理逻辑
- 显示运行日志
- 预览叠加结果图
- 打开输出目录

### 3.3 打包配置

文件：

- [CrackWidthInspector.spec](D:/PythonProjects/CrackWidthInspector/CrackWidthInspector.spec)
- [build_exe.ps1](D:/PythonProjects/CrackWidthInspector/build_exe.ps1)

说明：

- `spec` 文件将 `models/` 作为数据文件打包
- 使用目录版输出，提高稳定性
- GUI 程序关闭控制台窗口，更接近客户交付场景

## 4. 输入与输出

### 4.1 输入

- 单张图像：`jpg/jpeg/png/bmp/tif/tiff`
- 图像文件夹：递归扫描支持格式
- 比例尺参数：`scale(mm/px)`

### 4.2 输出

每张输入图像都会生成以下结果：

- `*_mask.png`：最终裂缝掩膜
- `*_skeleton.png`：裂缝骨架
- `*_overlay.png`：宽度标注叠加图
- `*_widths.csv`：所有骨架点宽度
- `*_profile.csv`：主路径沿程宽度剖面
- `*_samples.csv`：代表性采样点宽度

## 5. 依赖说明

### 5.1 运行依赖

- `numpy`
- `opencv-contrib-python`
- `PySide6`

- 可以优先使用 `ximgproc.thinning`

## 7. 局限性

- `scale` 未标定时，毫米值仅作估算
- 当前主路径分析只关注最大连通裂缝主支，不覆盖所有分叉
- 对极端低对比度、严重模糊、反光强烈的图像，结果仍可能退化
- HED 是通用边缘模型，并非混凝土裂缝专门训练模型
